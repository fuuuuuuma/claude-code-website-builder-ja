ループで完璧なWebサイトを作る
プロンプト＆キット

下のプロンプトはワンクリックでコピー、または.mdでダウンロードできます。キット全体は GitHub からどうぞ。

1壁打ちプロンプト（1発で作らせない）

いきなり作らせず、一問一答でサイトの仕様（site-spec.md）を固めるための最初の指示。

  • Codex/Claude Code の最初のメッセージとして貼る。
これからWebサイトを作りたい。ただし今すぐ作らないで。
まずプランモードで、一度に1問ずつ・各問に推奨案をつけて、私に壁打ちして。
最低でも「目的（1つ）／見せる相手（1人）／ゴール(CTA1つ)／必要なページ・セクション／
各ボタンの押した先／参考サイト／トンマナ／制約」を埋めて、site-spec.md にまとめてから見せて。
⬇ ダウンロード（.md）

💡 ここで目的とゴールを1つに絞れるほど、後のループが速く収束する。

2ループで各観点100点まで（サブエージェント採点）

固めた仕様から、複数のサブエージェントが採点しながら100点まで自走で作り込むループの起動プロンプト。

  • 壁打ちで site-spec.md を固めた後に送る（本キットのループ入口＝/site-goal）。
/site-goal site-spec.md のとおりのWebサイトを、3体以上のサブエージェントが各観点で100点を出すまで作り込む。
観点=機能／デザイン／モバイル&速度／コピー&法規制。採点はサブエージェントが0-100で行い、自分では採点しない。
「側だけ」で止めず、全ボタン・全リンク・全フォームの押した先まで実際に動かす（href="#"やダミーを残さない）。
デザインは1つの配色トークン(:root)に固定し、参考サイトのトンマナに寄せる。要素は足しすぎない(Simple is the best)。
全観点100点になるまで 実行→検証→修正 を繰り返し、なったら「GitHub Pages か Vercel か」を私に質問して公開まで。
採点は「ウルトラコード」（マルチエージェント並列）で回してよい。
⬇ ダウンロード（.md）

💡 採点は必ずサブエージェント（maker≠checker）。自分で「100点」と言わせない。

3「側だけ」チェック（押した先まで）

見た目だけの“ガワ”になっていないか、全導線を実機で押して確認させるプロンプト。

  • ビルド後・公開前にそのまま送る。
今のサイトの全リンク・全ボタン・全フォームを実機で1つずつ押して、
「押した先が存在するか・送信が成立するか・戻れるか」を確認して、
動かない導線とリンク切れを一覧で出して。href="#" が残っていたら全部指摘して直して。
⬇ ダウンロード（.md）

💡 href="#" やダミーが残っていないかが最重要。

4デザインをDESIGN.mdで固定

配色・フォントを1枚のDESIGN.mdに固定し、途中でブレさせないための指示。

  • ビルド開始時、または見た目がバラつくときに送る。
配色は design-systems の DESIGN.md を1枚選んで :root を固定して。途中で色を足さないで。
見出しフォントも DESIGN.md のものに。参考サイト（URL）のトンマナに寄せて、レイアウトはコピーしないで。
⬇ ダウンロード（.md）

💡 参考サイトはトンマナの当て先。レイアウトはコピーしない。

5画像だけ作らせる（GPT Image 2.0）

背景・装飾・OG画像などをAIに1枚生成させるプロンプト。

  • ヒーローや装飾に画像が要るときに送る。
OPENAI_API_KEY を設定して、scripts/gen_image.py（gpt-image-2）でヒーロー背景を1枚作って（横長・◯◯な雰囲気・文字は入れない）。
重要な見出し・価格・CTAは画像に焼き込まず、HTMLテキストで置いて。生成後に崩れを目視して。
⬇ ダウンロード（.md）

💡 OpenAI APIのgpt-image-2を使う（OPENAI_API_KEYが必要）。重要テキストは必ずHTMLに。

6公開（GitHub / Vercel・人間確認）

100点になったら公開先を選ばせ、本番でも動くか確認させるプロンプト。

  • 全観点100点になった後に送る。
全観点100点になったら、GitHub Pages と Vercel のどちらで公開するか私に聞いて。
選んだ先に公開したら、本番URLが200を返すか・本番でも全導線が動くかを確認して報告して。
（独自ドメイン・決済・フォーム実連携は、実行前に必ず私に確認して）
⬇ ダウンロード（.md）

💡 デプロイ・決済・外部連携は取り消せない操作。実行前に必ず人間が確認。

7ウルトラコードで徹底採点（Claude Code）

採点役をマルチエージェントで並列に回し、山場を一気に詰めるプロンプト。

  • 合格が渋いとき／公開前の総点検で送る。
採点を「ウルトラコード」（マルチエージェントのWorkflow）で並列に回して。
機能・デザイン・モバイル・コピーを同時に、各観点をさらに細かく分けて0-100で採点し、100点まで直して。
公開前に /code-review でも最終レビューして。使うのは山場だけ（トークンは多めなので）。
⬇ ダウンロード（.md）

💡 トークンは多め＝山場だけで使う。公開前は /code-review も併用。

キット全体を受け取る

壁打ち→ループ→採点→公開までを回す一式（スキル／コマンド／DESIGN.md 30選／参考資料）は、リポジトリからまるごと使えます。

GitHub でキットを開く キットの使い方（README）

受け取り方・使い方

  1. 使いたいプロンプトの「📋 コピー」を押す（または「⬇ ダウンロード」で .md 保存）。GitHub の kit/prompts/ からも受け取れます。
  2. Claude Code でこのキットのフォルダを開き、まず「壁打ちプロンプト」を貼る。
  3. 仕様が固まったら「ループ」プロンプトで各観点100点まで自走 → 公開まで。
⚠️ 使うときの注意：AIの出力（コピー・数値・リンク先）は確定前に必ずレビューを。公開・決済・外部連携などの取り消せない操作は、実行前に人間が確認してください。
