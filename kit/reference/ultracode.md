# ウルトラコードと Claude Code の並列マルチエージェント

Claude Code は、このキットのループを一番きれいに回せる道具を最初から持っています。
その中でも「採点役を一気に並列で走らせて、徹底的に検証する」使い方が **ウルトラコード（ultracode）** です。

> 出典の中心: Anthropic「[Getting Started with Loops](https://claude.com/blog/getting-started-with-loops)」。
> ここで挙がる `/goal`・サブエージェント・`/loop`・`/schedule`・スキル・**dynamic workflows** は Claude Code ネイティブ。
> 機能や呼び名は更新されるため、最新は Claude Code の公式ドキュメント／`/help` で確認してください。

---

## そもそも「ウルトラコード」とは

**マルチエージェントの Workflow（オーケストレーション）を明示的にオンにして使う、徹底モード**です。
1体のエージェントで順番に進める代わりに、**たくさんのサブエージェントを並列に走らせて**
「探す → 検証する → まとめる」を回します。網羅性・正しさを最優先する代わりに**トークンを多く使う**ので、
「軽い修正」ではなく「徹底的にやり切りたい」ときの選択肢です（**オプトイン**＝自分で使うと決めたときだけ）。

このキットのループ（maker ≠ checker）と相性が抜群です。**採点役の4観点を1体ずつ順番にやる**のではなく、
**機能・デザイン・モバイル&速度・コピーを"同時に"走らせ**、さらに各観点を細かく分けて検証できます。

## サイト作りでの使いどころ

| 場面 | ふつうのループ | ウルトラコード（並列・徹底） |
|---|---|---|
| QA採点 | 4観点を順番に採点 | 4観点＋細分（リンク/フォーム/速度/A11y…）を**並列**で一気に採点 |
| 参考サイト調査 | 1本ずつ読む | 複数ギャラリーを**並列**で横断して当て先を探す |
| 最終レビュー | 目視で確認 | `/code-review`（ウルトラ版はマルチエージェントのクラウドレビュー）で観点別に総点検 |

> 使いすぎ注意：**小さな修正に多段の並列は過剰**。公式ブログの原則どおり「まず一番簡単な解から」。
> 徹底したい山場（QA100点まで／公開前レビュー）でだけ使うのがコツ。

## Claude Code のループ道具（このキットで使うもの）

- **`/goal`（このキットでは `/site-goal`）** — 成功条件（各観点100点）を決めて、達成か上限まで自走する goal-based loop。
- **サブエージェント（`Task`）** — 別の"新鮮な目"を召喚して採点させる。**maker ≠ checker** の要。並列で複数同時に。
- **ウルトラコード（Workflow）** — 上記を**マルチエージェントで並列オーケストレーション**する徹底モード（この文書の主役）。
- **スキル（`SKILL.md`）** — 「良いとは何か」「検証手順」を型として教える（このキットの `site-grilling`／`site-builder`）。
- **フック（hooks）** — 保存時に整形・チェックを自動実行する等、"毎回やること"を自動化。
- **`/code-review`** — 変更点をレビュー。**ウルトラ版はブランチ/PRをマルチエージェントでクラウドレビュー**（最終点検に）。
- **`/loop`・`/schedule`** — 時間・間隔で回す（公開後の定期チェックや監視に）。

## 使い方（かんたん）

1. まずは普通に `/site-goal` で回す（3体のサブエージェント採点）。多くのサイトはこれで十分。
2. **もっと徹底したい山場**（合格が渋い・公開前の総点検）で、マルチエージェントを並列に走らせる＝ウルトラコード。
   - 「採点を**並列マルチエージェント**でやって。機能・デザイン・モバイル・コピーを同時に、各観点をさらに細分して100点まで」
3. 公開前に **`/code-review`** で最終レビュー。指摘があればループに戻す。

> どれも「合格の定義（数えられるチェック）」と「採点役を分ける（maker ≠ checker）」という
> ループの2原則を、Claude Code の道具で徹底しているだけです。難しく考えず、山場で使ってください。
